========================================
尝试功能实现和设计 aided by DeepSeek
----------------------------------------
不依赖后端，怎么在前端实现英语口语练习，要求逐句播放，对用户跟读进行录音，并提供点击播放原始录音和点击播放跟读录音的功能，最好比对峰值差异

1. 功能需求
逐句播放：播放原始音频的每一句。
录音：录制用户的跟读音频。
播放原始录音：点击播放原始音频。
播放跟读录音：点击播放用户录制的音频。
峰值比对：可视化原始音频和跟读音频的波形，并比对差异。



2. 技术实现
使用以下技术和 API：

Web Audio API：用于音频分析和波形绘制。
MediaRecorder API：用于录制音频。
HTML5 Audio：用于播放音频。
Canvas：用于绘制音频波形。

4. 功能说明
逐句播放：通过 Audio 对象播放原始音频。
录音：使用 MediaRecorder 录制用户音频。
播放录音：通过 Audio 对象播放录制的音频。
波形比对：使用 Web Audio API 分析音频数据，并通过 Canvas 绘制波形。



5. 峰值比对
通过绘制原始音频和跟读音频的波形，可以直观地比对两者的峰值差异。如果需要更精确的比对，可以计算波形的均方误差（MSE）或其他相似性指标。


6. 注意事项
浏览器兼容性：确保使用支持 MediaRecorder 和 Web Audio API 的浏览器（如 Chrome、Firefox）。
性能优化：对于较长的音频，可能需要优化波形绘制逻辑。




7.报错
(1) script.js:55 The AudioContext was not allowed to start. It must be resumed (or created) after a user gesture on the page. https://goo.gl/7K7WLu
这个错误是由于浏览器的自动播放策略导致的。从 Chrome 66 开始，浏览器要求音频上下文（AudioContext）必须在用户与页面交互（如点击按钮）后才能启动。这是为了防止网页自动播放音频，影响用户体验。

你需要在用户与页面交互（如点击按钮）后，手动恢复或创建 AudioContext。


(2) navigator.mediaDevices is null
https://stackoverflow.com/questions/60957829/navigator-mediadevices-is-undefined
