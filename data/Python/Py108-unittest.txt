单元测试：相关py包，及概念
----------------------------------------
1.测试包
(1).unittest
py内置包。


(2).pytest
Third-party unittest framework with a lighter-weight syntax for writing tests. For example, assert func(10) == 42.


2.测试相关概念
(1)test fixture 测试固件: 用于设置多个测试状态的类，通常用于设置“代价昂贵”或执行时间过长时使用。
A test fixture represents the preparation needed to perform one or more tests, and any associated cleanup actions. This may involve, for example, creating temporary or proxy databases, directories, or starting a server process.


(2)test case 测试实例
A test case is the individual unit of testing. It checks for a specific response to a particular set of inputs. unittest provides a base class, TestCase, which may be used to create new test cases.

(3)test suite 测试实例的集合
A test suite is a collection of test cases, test suites, or both. It is used to aggregate tests that should be executed together.


(4)test runner
测试运行器是一个组件，它编排测试的执行并向用户提供结果。运行程序可以使用图形界面、文本界面或返回一个特殊值来指示执行测试的结果。
A test runner is a component which orchestrates the execution of tests and provides the outcome to the user. The runner may use a graphical interface, a textual interface, or return a special value to indicate the results of executing the tests.

unittest 入门
----------------------------------------
1.入门案例

(1)代码
import unittest

class TestStringMethods(unittest.TestCase):

    def test_upper(self):
        self.assertEqual('foo'.upper(), 'FOO')

    def test_isupper(self):
        self.assertTrue('FOO'.isupper())
        self.assertFalse('Foo'.isupper())

    def test_split(self):
        s = 'hello world'
        self.assertEqual(s.split(), ['hello', 'world'])
        # check that s.split fails when the separator is not a string
        with self.assertRaises(TypeError):
            s.split(2)

if __name__ == '__main__':
    unittest.main()

(2)执行
$ python3 tests/a02_test_str.py 
...
----------------- ----------------------- --------------- ---------------
Ran 3 tests in 0.000s

OK

第一行3个点，表示三个函数执行结果符合预期。

* 使用 -v 参数能给出更详细的输出



(3)函数解释
继承 unittest.TestCase 类，其中的方法名开头是 test，表示要测试的函数。

- assertEqual(): 检查期望值
- assertTrue() or assertFalse()  检查是否符合条件
- assertRaises() 验证是否引发了特定的异常

用以上代替 assert 语句，来执行所有测试，并给出报告。

- setUp() and tearDown() 方法可以在执行test前后设置一些变量。
	详情见: https://docs.python.org/3/library/unittest.html#organizing-tests

- unittest.main() 运行测试。

1.基础是 test case
* 每个 TestCase 子类必须自包含，方便随意组合运行测试。
* 使用 TestCase 提供的  assert* methods 方法测试具体条件。

import unittest

class DefaultWidgetSizeTestCase(unittest.TestCase):
    def test_default_widget_size(self):
        widget = Widget('The widget')
        self.assertEqual(widget.size(), (50, 50))

如果测试失败，就会产生一个异常，其他异常会被视为错误。






2.设置条件 setUp()
测试可能很多，设置也是重复的。
每个简单test前，都会调用 setUp() 方法。

import unittest

class Widget():
    name=""
    wide=50
    height=50
    def __init__(self, name):
        self.name=name
    def size(self):
        return (self.wide, self.height)
    def resize(self, wide, height):
        self.wide=wide
        self.height=height


class WidgetTestCase(unittest.TestCase):
    def setUp(self):
        self.widget = Widget('The widget')

    def test_default_widget_size(self):
        self.assertEqual(self.widget.size(), (50,50),
                         'incorrect default size')

    def test_widget_resize(self):
        self.widget.resize(100,150)
        self.assertEqual(self.widget.size(), (100,150), # 尝试修改为(100, 151)试试，会出现一个F
                         'wrong size after resize')
if __name__ == '__main__':
    unittest.main()
# $ python3 -m unittest tests/a03_test_setUp.py

- 如果 setUp() 执行报错，就不执行测试代码。



3. tearDown() 方法在每个方法后执行清理工作
(1)
import unittest

class WidgetTestCase(unittest.TestCase):
    def setUp(self):
        self.widget = Widget('The widget')

    def tearDown(self):
        self.widget.dispose()

(2) 如果setUp()执行成功，不管测试结果，一定会执行 tearDown() 清理工作
If setUp() succeeded, tearDown() will be run whether the test method succeeded or not.


(3) 测试代码运行的环境被称为 test 固件
Such a working environment for the testing code is called a test fixture.

每个test方法的执行都创建一个独特的 测试固件，方便 setUp, tearDown, TestCase.__init__ 执行。
A new TestCase instance is created as a unique test fixture used to execute each individual test method. Thus setUp(), tearDown(), and TestCase.__init__() will be called once per test.

|-- 自定义测试 TestSuite()
----------------------------------------
1.示例
def suite():
    suite = unittest.TestSuite()
    suite.addTest(WidgetTestCase('test_default_widget_size'))
    # suite.addTest(WidgetTestCase('test_widget_resize'))
    return suite

if __name__ == '__main__':
    runner = unittest.TextTestRunner()
    runner.run(suite())

没发现有什么用，我注释掉一行，还是执行2个测试！！

原因: 当 unittest 发现一个 TestCase 类时，默认会执行该类中所有以 test 开头的方法。即使你的 TestSuite 只添加了一个方法，unittest 可能在其他地方（如 unittest.main() 或自动发现机制）也会运行整个测试类。


(1)方式1: 通过 unittest 模块运行（你使用的方式）
会执行所有 test_* 方法，忽略 suite()
$ python3 -m unittest tests/a03_test_setUp.py #执行2个方法
要去掉 -m unittest 参数


(2)方式2：直接运行 Python 文件
会执行 __main__ 中的代码，使用你定义的 suite() #执行1个方法
$ python3 tests/a03_test_setUp.py




2.TestSuite 的真正用途
- 控制执行顺序和内容
- 选择性执行测试
- 组合多个测试类
- 动态构建测试套件
- 控制测试执行顺序
