========================================
win dos: graphics.h
----------------------------------------
1. 安装libgraph库
在Ubuntu系统中使用graphics.h需要借助第三方库实现兼容性。
(1)安装依赖项
sudo apt update
sudo apt install build-essential libsdl-image1.2-dev guile-2.0-dev libfreetype6-dev libxext-dev

(2)下载并编译libgraph‌
从官网下载源码包（如libgraph-1.0.2.tar.gz）。
解压并进入目录后执行：

./configure
make
sudo make install


(3)‌验证安装‌
编写测试程序，编译时需链接库

$ cat test.c
#include <graphics.h>
int main() {
    int gd = DETECT, gm;
    initgraph(&gd, &gm, "");
    circle(320, 240, 100);
    getch();
    closegraph();
    return 0;
}

$ gcc test.c -o test -lgraph




2.替代方案
‌(1)SDL_bgi或libXbgi‌
这两个库为现代Linux环境提供了更稳定的图形接口，安装方法：

sudo apt install libsdl-bgi-dev  # SDL_bgi
# 或从源码编译libXbgi

‌(2)模拟器运行Turbo C‌
通过Bochs或DOSBox模拟器安装Turbo C，直接使用原生graphics.h




3.常见问题
‌编译报错“graphics.h not found”‌：需确认libgraph头文件是否安装到/usr/local/include，或手动指定头文件路径‌。
‌链接错误‌：编译时需添加-lgraph参数，若库路径未识别，可尝试-L/usr/local/lib
