    2. IDE选择

  2. MyEclipse和Eclipse哪个好？
    如果喜欢eclipe，那就直接用eclipse for j2ee吧，如果你不在乎盗版，而且还要求更好的体验那么请选择Intellij IDEA。至于myeclipse，实在是没有任何理由让我使用它。
    用最新版的吧，我用的myeclipse10，现在这个版本可以把不用的组件卸载，速度快一点。
    eclipse jee版也挺好用的，支持tomcat这些服务器的最新版，以及JDK最新版。

    至于SSH这些框架的最新版，倒是不推荐过于依赖IDE，把该用的jar和配置文件弄好就行。

    3. 语言深度和广度

  3. 什么算精通java
    我理解的精通>
    1 能在实践中深刻体会过这门语言的优势和弊端，知道它的能力边界
    2 每一次更新，你会主动关心新特性，思考会对以后项目实践有何影响
    3 你运用这门语言解决问题的能力，超过大多数人
    4 当你真正觉得语言就是个工具的时候，看到问题想到的是架构，模型，算法等，能通过业务需求选择需要，选择用java还是其他的语言时，就算你最终选择弃用JAVA，我想你也是精通它了

  6. 现在还适合学Java吗？
    支持学Java：
    后端语言当然首选 Java 啦，现在 SpringBoot 很简单了，而且你学了 Java 就了解静态语言的魅力,了解为何大公司都用 Java。
    如果是打算近期往后端开发转，那么 java 互联网第一语言还是首选，如果只是有点服务端开发的需求需要处理，那么可以尝试下 go，部署运维简单

    反对学Java：
    现在学 java ？找不自在么？人那么多，封装的那么深，学的都不是语言，而是学框架了。而且还挺复杂，学不完。
    python 方便。java 每次编译，烦死了。
    python 学一个月，java 学一年，你自己选
    python 入门比较友好，而 java 比较重，学习成本高
    Python + Flask，一小时上手。反正又不要做生产环境运行的正式后端。千万别选 Django，只是为了配合 JS 前端的话，那东西太笨重。
