(1) 卷积层 convolutional layer:

Convolutional layers, which apply a specified number of convolution filters to the image. For each subregion, the layer performs a set of mathematical operations to produce a single value in the output feature map. Convolutional layers then typically apply a ReLU activation function to the output to introduce nonlinearities into the model.

卷积层，就是使用一系列filter对图片做卷积计算，提取特征。并且典型的卷积层会使用ReLU激活函数来引入非线性因数。

卷积的计算过程方法如下图，也就是用filter扫描整个图片，并做wx+b计算：


一般要用激活函数，如 ReLu，引入非线性。
引入非线性的作用：
  a)减少计算 b)避免梯度消失 c)网络的稀疏性，避免过拟合
  引入非线性是为了提取特征的，线性变换的结果还是线性的，没有意义的。


(2) Pooling layers 池化层
池化这个概念相对简单，类似于压缩图片。
max pool即取设定矩形里最大的值作为输出数据。


(3) Dense(fully connected) Layer 稠密层(全连接层) 

通过卷积和池化，然后得到了众多特征，稠密层的每一个节点都与这些特征节点相连构成稠密层(全连接层)。

稠密层的作用就是分类。

简单的说就是每一个特征节点手里握着一定的权重来决定输入是属于那个分类，最终全部特征的权重共同决定了输入所属分类的权重或概率。

也有反对意见: dense 应该不是分类，它的作用是将所有的数据连接在一起，说白了就是把所有的分开的小的感受野组合成整体的感受野。
分类一般是用后面的softmax来实现的。

|-- --浅谈神经网络中的bias
----------------------------------------
1、什么是bias？

偏置单元（bias unit），在有些资料里也称为偏置项（bias term）或者截距项（intercept term），它其实就是函数的截距，与线性方程 y=wx+b 中的 b 的意义是一致的。在 y=wx+b中，b表示函数在y轴上的截距，控制着函数偏离原点的距离，其实在神经网络中的偏置单元也是类似的作用。 
因此，神经网络的参数也可以表示为：(W, b)，其中W表示参数矩阵，b表示偏置项或截距项。




2.bias的计算方式？
神经网络结构中对偏置单元的计算处理方式有两种， 
(1) 略 非主流

(2) 设置偏置单元，不在参数矩阵中设置对应偏置单元的参数，对应的神经网络如下： 
输入 [x1,x2,x3]T 添加常数项 [b1, x1,x2,x3]T

其中，b(1) 是 W(1) 对应的偏置单元向量，b(2) 是 W(2) 对应的偏置单元向量，b(1)1 是对应 a(2)1 的偏置单元。注意，此时神经网络的参数表示更改为：(W, b) 
* 在计算激活值时按照： 
激活值计算公式 a2=g(W1*x1 + w2*x2 + w3*x3 + b1)

相当于bias连接各个神经元的所有权重都为1，但bias本身不为1，即---有多个bias，但所有的bias对应的权重都为1（bias的个数等于hide层和out层神经元的个数）


综上， 
　两者的原理是一致的，只是具体的实现方式不同。 
　其实在大部分资料和论文中看到的神经网络的参数都是表示为：(W, b)，其中W代表weight，b代表bias。包括在UFLDL Tutorial中也是采用 (W, b) 表示，只是在Coursera上Andrew Ng老师的在线教程中看到将神经网络参数表示为 Θ，个人还是更喜欢 (W, b) 这种表示，很清晰。

熵(Entropy)，交叉熵(cross Entropy)，损失函数(loss function)
----------------------------------------

小结：交叉熵的“交叉”指的是真实分布和估计的交叉，并且希望这个值越小越好，所以可以用来做损失函数。

1. 香农提出了熵的定义：无损编码事件信息的最小平均编码长度。

P=1/N;
log2(N)=-log2(1/N)=-log2(P)

那么熵的那些描述和解释(混乱程度，不确定性，惊奇程度，不可预测性，信息量等)代表了什么呢？

如果熵比较大(即平均编码长度较长)，意味着这一信息有较多的可能状态，相应的每个状态的可能性比较低；因此每当来了一个新的信息，我们很难对其作出准确预测，即有着比较大的混乱程度/不确定性/不可预测性。

并且当一个罕见的信息到达时，比一个常见的信息有着更多的信息量，因为它排除了别的很多的可能性，告诉了我们一个确切的信息。在天气的例子中，Rainy发生的概率为12.5%，当接收到该信息时，我们减少了87.5%的不确定性(Fine,Cloudy,Snow)；如果接收到Fine(50%)的消息，我们只减少了50%的不确定性。

E(x)=累加(i, -pi*log2(pi))

定义R函数：
entropy=function(arr){
  if(1!=sum(arr)){stop("sum not 1")}
  results=0
  for(p1 in arr){
    results = results -p1*log2(p1)
  }
  results
}

仅有一个分类时，熵最小，为0.
仅有2个分类时，熵最大为平分：
	> entropy(c(0.5, 0.5)) #[1] 1
	> entropy(c(0.1, 0.9)) #[1] 0.4689956

其中：
	> -0.1*log2(0.1) #	[1] 0.3321928  小概率时间对应的熵大
	> -0.9*log2(0.9) #	[1] 0.1368028

分组越多，最大熵越大，就是平分的时候：
	> entropy( rep(1/3, 3) ) #[1] 1.584963
	> entropy( rep(1/4, 4) )# [1] 2
	> entropy( rep(1/5, 5) )# [1] 2.321928
	> entropy( rep(1/10, 10) ) #[1] 3.321928





2. 交叉熵

(1) 另一种写法

E(x)=累加(i, -pi*log2(pi))

x~P代表我们使用概率分布P来计算期望，熵又可以简写为H：

H(P)=Entropy=Ex~p[-logP(x)]

重要的事情再说一遍：“熵是服从某一特定概率分布事件的理论最小平均编码长度”，只要我们知道了任何事件的概率分布，我们就可以计算它的熵；那如果我们不知道事件的概率分布，又想计算熵，该怎么做呢？那我们来对熵做一个估计吧，熵的估计的过程自然而然的引出了交叉熵。




(2) 熵的估计

假如我们现在需要预报东京天气，在真实天气发生之前，我们不可能知道天气的概率分布；但为了下文的讨论，我们需要假设：对东京天气做一段时间的观测后，可以得到真实的概率分布P。

在观测之前，我们只有预估的概率分布Q，使用估计得到的概率分布，可以计算估计的熵：

EstimatedEntropy = Ex~Q[-logQ(x)]

如果Q是真实的概率分布，根据上述公式，我们已经得到了编码东京天气信息的最小平均长度；然而估计的概率分布为我们的公式引入了两部分的不确定性：
* 计算期望的概率分布是Q，与真实的概率分布P不同。
* 计算最小编码长度的概率是 -logQ，与真实的最小编码长度 -logP 不同。

因为估计的概率分布Q影响了上述两个部分(期望和编码长度)，所以得到的结果很可能错得离谱，也因此该结果和真实熵的对比无意义。和香农的目标一样，我们希望编码长度尽可能的短，所以我们需要对比我们的编码长度和理论上的最小编码长度(熵)。假设经过观测后，我们得到了真实概率分布P，在天气预报时，就可以使用P计算平均编码长度，实际编码长度基于Q计算，这个计算结果就是P和Q的交叉熵。这样，实际编码长度和理论最小编码长度就有了对比的意义。

CrossEntropy = Ex~P[-logQ(x)]

Entropy=Ex~P[-logP(x)]




(3) 交叉熵 >= 熵

交叉熵使用H(P,Q)表示，意味着使用P计算期望，使用Q计算编码长度；
所以H(P,Q)并不一定等于H(Q,P)，除了在P=Q的情况下，H(P,Q) = H(Q,P) = H(P)。

H(P,Q)=Ex~P[-logQ(x)]

有一点很微妙但很重要：
* 对于期望，我们使用真实概率分布P来计算；
* 对于编码长度，我们使用假设的概率分布Q来计算，因为它是预估用于编码信息的。

因为熵是理论上的平均最小编码长度，所以交叉熵只可能大于等于熵。

换句话说，如果我们的估计是完美的，即Q=P，那么有H(P,Q) = H(P)，否则，H(P,Q) > H(P)。

至此，交叉熵和熵的关系应该比较明确了，下面让我们看看为什么要使用交叉熵作分类损失函数。




(4) 交叉熵作为损失函数
假设一个动物照片的数据集中有5种动物，且每张照片中只有一只动物，每张照片的标签都是one-hot编码。

Label Animal
10000 Dog
01000 Fox
00100 Horse
00010 Eagle
00001 Squirrel


第一张照片是狗的概率为100%，是其他的动物的概率是0；
第二张照片是狐狸的概率是100%，是其他动物的概率是0，
其余照片同理；
因此可以计算下，每张照片的熵都为0。
换句话说，以one-hot编码作为标签的每张照片都有100%的确定度，不像别的描述概率的方式：狗的概率为90%，猫的概率为10%。


假设有两个机器学习模型对第一张照片分别作出了预测：Q1和Q2,而第一张照片的真实标签为[1,0,0,0,0]。
Model Prediction
Q1    [0.4,0.3,0.05,0.05,0.2]
Q2    [0.98,0.01,0,0,0.01]


两个模型预测效果如何呢？可以分别计算交叉熵：

H(P1,Q1) = -累加(Pi*log2(Qi)) = -( 1*log2(0.4)+0*log2(0.3)+0*log2(0.05)+0*log2(0.05)+0*log2(0.2) )=1.3219
H(P1,Q2) = -累加(Pi*log2(Qi)) = -( 1*log2(0.98)+0*log2(0.01)+0*log2(0)+0*log2(0)+0*log2(0.01) )=0.029

交叉熵对比了模型的预测结果和数据的真实标签，随着预测越来越准确，交叉熵的值越来越小，如果预测完全正确，交叉熵的值就为0。
因此，训练分类模型时，可以使用交叉熵作为损失函数。




(5) 二分类交叉熵
在二分类模型中，标签只有是和否两种；这时，可以使用二分类交叉熵作为损失函数。假设数据集中只有猫和狗的照片，则交叉熵公式中只包含两种可能性：

H(P,Q)=-累加(i=(cat,dog), Pi*logQi) = -P(cat)logQ(cat) - P(dog)logQ(dog)

又因为 P(dog)=1-P(cat)
所以交叉熵可以表示为：

H(P,Q)= -P(cat)logQ(cat) - (1-P(cat))log(1-Q(cat))

使用如下定义
	P=P(cat)
	P_bar=Q(cat)

二分类的交叉熵可以写作如下形式，看起来就熟悉多了。

BinaryCrossEntropy=-Plog(P_bar) - (1-P)log(1-P_bar)

|-- 损失函数(loss function) 及其求解
----------------------------------------
0.引入问题: 从函数到神经网络。
符号派认为所有问题都可以找到规律
[0,1,2,3] => [0,2,4,6] 就是 y=2*x
如果不能一眼看明白呢？需要稍作计算：

(1)线性问题拟合 y=a.x + b
参数a也叫做x的权重，记作w。

(2)为了拟合线性问题，外面嵌套一层 非线性函数，叫激活函数
y=g( w.x + b)

常见的激活函数都十分简单: 4种，ReLu, Sigmoid 最常用。

(3)输入变量可能是多个，对应的权重也是多个
f(x1, x2)=g(w1.x1 + w2.x2 + b)

(4)然后这个可以作为整体，再嵌套一层权重w3, b2，及激活函数g2
g2( w3( g(w1.x1 + w2.x2 + b) ) + b2 )
这样不断套娃，找到合适的w和b，就能预测所有问题。


(5)这个嵌套模式展开，就是神经网络
*
*  *   *
*  *   *
*  *
*
输入层 -w-> g(隐藏层) -w2-> g2(输出层)




1.衡量预测值和真实值的差异的函数：损失函数
什么是好的w,b？预测值和真实值越接近，拟合的就越好。
就是损失函数最小。对损失函数求导，求极致处的w和b。

(1)数学化:
1)模仿线性拟合，对每个x，其拟合值 y_hat 和 真实值 y 的差异(沿着x画竖线)的和: sum( |yi-yi_hat| )
2)绝对值不好做数学优化，为了方便计算导数，同时放大误差较大的极端值的贡献，改绝对值为平方 (yi-yi_hat)^2
3)为了消除样本量N对误差的影响，除以N，得到 均方误差(MSE)=1/N * Sigma(i=1, N, (yi-yi_hat)^2 )
这就是常用的损失函数(Loss, L 函数)中的一种了:
	L(w, b) = 1/N * Sigma(i=1, N, (yi-yi_hat)^2 )

从参数上理解，损失函数就是一个关于w和b的函数。

再回顾目的：求解让Loss最小的w和b的值。
	表示真实值与预测值的误差。

4) 怎么求解呢？导数为0，求极值点。



(2)求解示例：假设b=0
最简单的[x,y]: (1,1) (2,2) (3,3) (4,4)，求 y=wx 的权重w

L(w)=1/4 * ( (1-1x)^2 + (2-2x)^2 + (3-3x)^2 + (4-4x)^2 )
L(w)=7.5 -15w +7.5w^2
L'(w) = -15 + 15w=0
得到 w=1
所以：y=x就是让损失函数L最小的那条直线。

回顾 L(w)，是一个二次函数，开口向上的抛物线，最底部x=1是最低点。


(3)求解示例: b也未知
如果不知道之前的b=0呢？那就有两个未知数w和b。
这时的损失函数是一个开口向上的碗状曲面。
	L(w,b)=1/N.Sigma( yi - yi_hat )^2
这就要求偏导数:
	∂L/∂w=0 就是把b看作常数，只对一个未知数w求导。
	∂L/∂b=0 同上。

这就叫做 线性回归。



(3)对于非线性的更复杂情况，怎么求w和b呢？
简单粗暴的：分别调整w和b，检查L(w,b)的值是否变小了。
	对w分别变大和缩小1，看L是否变小了。 这个量化后，L变化/w的变化，就是偏导数 ∂L/∂w。
	对b同理：∂L/∂b
	偏导数构成的向量，叫做梯度。

更快速的：把w和b分别按照其偏导数的反方向变化：
	w=w-∂L/∂w
	b=b-∂L/∂b
	沿着梯度反方向调整系数w和b。

实践中发现，有些情况下这样变化太快，需要降低变化速度，再添加一个系数η，盖章又叫学习率。
	w=w - η*∂L/∂w
	b=b - η*∂L/∂b
	每次调整幅度降低为η倍。

这种沿着梯度的反方向，逐步调整w和b，最终求得Loss最小值时的参数w和b的过程，叫做 梯度下降法。



(4)现在的问题是怎么求偏导数？特别是整个函数特别复杂。
最简单的举例子 3 层: x -> a -> y
	有输入逐步计算出输出的过程：前向传播。

x ->g(w1.x + b1)->  a  ->  g(w2.a + b2) -> y_hat -> (y - y_hat)^2 -> L
	这里损失函数用 均方误差。
	只有1组数据，所以损失函数也省略了。

这个神经网络种共有4个参数需要求解：w1,b1, w2,b2
	使用梯度下降法。
	问题是如何求偏导数。
	∂L/∂w1 就是w1变化一点点，导致L变化多少。
	可以沿着链条逐步解释: 
		w1变化单位1，导致a变化多少？   ∂a/∂w1
		a变化单位1，导致y_hat变化多少？ ∂y_hat/∂a
		y_hat变化单位1，导致L变化多少？ ∂L/∂y_hat
	就是三个偏导数相乘，倒着放：∂L/∂w1 = ∂L/∂y_hat * ∂y_hat/∂a * ∂a/∂w1
		形象思考：几个大小不一的齿轮，第一个转了一圈，第三个转了几圈？

i)这种求导数的方法就叫做：链式法则(the chain rule)
	其实就是微积分种的 复合函数求导。

ii)由于我们可以从右向左，从输出到输入，逐步更新参数，直到把所有的 神经网络参数 都更新一遍。
	在前面(最右侧)计算的偏导数，后面(最左侧)还会用到，所以把这个从右向左传播过程叫做：反向传播。

iii)数据从input到output，获得一次预测值，然后根据梯度反方向调整参数w，这个过程叫一次训练。
	input -> pred ~ real  -->前向传播
	反向传播 <----- 沿着梯度反方向更新参数

总结:
	前向传播算输出；
	反向传播变参数；
	一个来回一次训练。

|-- 调教神经网络的方法：防止过拟合，采取正则化方法（L1, L2, Dropout）
----------------------------------------
1.我们似乎能找到最佳的参数w和b，预测任何问题了！
等等，没那么简单！

(1)过拟合
如果w和b“记住”了训练数据，表现很好，而对于 没见过的测试数据 表现很差，这就叫“过拟合”

(2)泛化能力
如果对训练数据效果好，对没见过的测试数据效果也好，这就叫做模型的“泛化能力”强。

(3)过拟合是因为模型太复杂了。
怎么让模型知道数据的规律，而不是记住数据点呢？
1).简化模型。说明神经网络不是越大越好。
2).增加训练数据量。

(4)鲁棒性
如果无法获取更多样本，可以制造一些略微改动的增强版样本，这样模型不会因为模型的微小变化而导致预测变化。
这叫增强了模型的“鲁棒性(Robustness)”

比如，对输入的dog照片，做旋转、翻转、裁切、灰度、调亮等操作。


(5)防止过拟合的另一个简单方法：提前终止。
就是不用训练那么完美，差不多就行了。这样反而泛化能力更强，更实用。


(6)正则化
提前终止太粗糙了，有没有更精细的办法抑制参数的过快增大？
另一种防止过拟合的方法是：把参数本身添加到损失函数中，如果参数增大后，损失没有降低，则新损失函数整体会增大。

相当于添加了一个 惩罚项(Penalty)：参数绝对值和，或者参数的平方和。
来抑制参数过快增长的方法，叫做 正则化。
	NewLoss=损失函数 + Sigma(i=1, N, |wi| ) #L1 惩罚项: L1 Penalty
	NewLoss=损失函数 + Sigma(i=1, N, wi^2 ) #L2 惩罚项: L2 Penalty

分别使用L1范数和L2范数作为惩罚项，所以也叫做L1正则化，L2正则化。


(7)正则化系数 lambda
为了防止该惩罚项过大，给惩罚项添加一个系数，叫 正则化系数 λ。
	NewLoss=损失函数 + λ*Sigma(i=1, N, |wi| ) #L1 惩罚项: L1 Penalty
	NewLoss=损失函数 + λ*Sigma(i=1, N, wi^2 ) #L2 惩罚项: L2 Penalty

该超参数主要是为了抑制参数的野蛮增长。

(8) 超参数
这些控制参数的参数，叫做超参数。

超参数的选择，是个玄学。主要靠经验和直觉。


(9)Dropout 丢弃：另一种避免过拟合的方法。
为了防止参数中某些参数过于厉害，导致整体预测只依靠这几个关键参数，导致过拟合，我们可以在训练中随机丢弃一些参数，让模型更多的依赖普通参数起作用。

这个方法是Hinton 提出的。



小结：怎么避免过拟合？
	增加数据量
	减少模型复杂度：
	提前终止训练
	L1正则化、L2正则化、Dropout



2.模型的其他问题，及对应措施
(1)梯度消失：模型越深，梯度反向传播时会越来越小，导致参数更新困难。
	残差网络：防止深层网络的梯度衰减。
(2)梯度爆炸：梯度数值越来越大，参数的调整幅度失去控制
	梯度裁剪：防止梯度更新过大。
	权重初始化、归一化，让梯度分布更平滑。
(3)收敛速度过慢：可能陷入局部最优解或者来回震荡。
	动量法、RMSProp、Adam等自适应优化器来加速收敛，减少震荡。
(4)计算开销过大：数据规模量太庞大，导致每次前向传播和反向传播都非常耗时。
	用 mini-batch 把数据分割成几个小批次，来降低单次的计算开销。

这些名词都是一个研究方向，都是为了解决模型的问题。这里略过细节。

参数太多，导致AI似乎变成了一门调参玄学，“数字炼丹”。

|-- 从矩阵到CNN(卷积神经网络)
----------------------------------------
资料

1.神经网络
(1)最简单的情况入手
x --> y

y=g(w.x + b)

(2)如果输入是两个，及更多
x1 --> y1  这些箭头从不同的x都指向同一个 y
x2 -->

y1=g(w1.x1 + w2.x2 + b1)

更多的x对应更多的w

(3)如果输出也是多个，那么公式也对应多个

x1 --> y1
x2 --> y2
x3 -->
箭头从每个x指向每个y

y1 = g( w11.x1 + w12.x2 +w13.x3 + b1 )
y2 = g( w21.x1 + w22.x2 +w23.x3 + b2 )

每个权重能区分开即可。
	比如w12表示第1个神经元的第2个参数。

这样似乎参数过多，能否简洁点？



2.矩阵形式
(1)把上述公式写成矩阵形式，先忽略激活函数

[y1 y2].T = 
	[w11 w12 w13]
	[w21 w22 w23].[x1 x2 x3].T + [b1 b2].T

左边矩阵相乘得到 2行1列 矩阵。

(2)使用字母简化
Y=g(w.X + b)
其中:
	Y=[y1 y2].T

	W=[w11 w12 w13]
	  [w21 w22 w23]

	X=[x1 x2 x3].T

	b=[b1 b2].T

(3)对于多层神经网络
前一层是输入：A[L-1]
下一层是输出: A[L]

	对于网络 a[0] a[1] a[2] ... a[L-1] a[L]
	公式: A[L]=g(w[L].X[L-1] + b[L])

好处:
	更简洁，更抽象，可以进行更高级、更复杂的描述。
	把简单运算抽象为矩阵运算，可以使用GUP硬件加速计算。加速训练和推理过程。



3.全连接层
(1)每一个神经元都和前一层的所有神经元连接，这叫做全连接层(Fully-Connected Layer)

	输入图像：30*30
	伸直：900
	下一层：1000
	则权重: 90万参数

全连接层的缺陷：太大，无法保留图像的空间关系。


(2)卷积层 conv
如果固定一个矩阵3*3，和原始图形的 30*30 的每个3*3像素运算：对应元素相乘再求和。这就是卷积。
求卷积得到新矩阵的层，叫卷积层。

这个固定矩阵，叫卷积核。
卷积核是此前图像处理领域早已使用的，比如模糊效果、浮雕效果、轮廓效果、锐化效果等PS常规操作。

而神经网络中，卷积核的值是未知的，是训练出来的一组数值。

把刚才的神经网络中的某个全连接层替换为卷积层，就大大减少了参数数量，同时更能捕捉到图片中的一些局部特征。

公式上看：就是把之前的w.A矩阵运算，替换为卷积运算。
	原来：A[L]=g(w[L].X[L-1] + b[L])
	现在：A[L]=g(w[L]*X[L-1] + b[L])

接下来的神经网络图示，每一层就不画圈了，只用矩形表示一个层。
	输入层  ->  卷积层 -> 全连接层 -> 输出层
     input      conv       FC          output

卷积层后通常还有池化层(Pooling Layer)，作用是对卷积后的图像进行降维。
	作用是减少计算量，同时保留主要特征。
	输入层  ->  卷积层 -> 池化层 -> 全连接层 -> 输出层
     input      conv      pool      FC          output

一个网络中可以有多个这些层：卷积层、池化层、全连接层，这就是图像识别领域的实际网络结构。
	input conv pool conv pool FC FC output

含有卷积层的神经网络，叫卷积神经网络 CNN。


=> 矩阵计算的简单直观方法：A.B = 把B向上推出，则留下的空间，就是A.B矩阵乘法的维度：左边的行，上边的列。

小结:
	神经网络多输入和多输出，可以写成多个公式。
	这个多元一次方程，可以写成矩阵形式 Y=g(W.X + b)

	全连接层 有局限性，添加卷积层、池化层后，获得卷积神经网络。

	神经网络图示更简化：每一层有N个圆圈表示神经元个数，简化为一层都是一个矩形。


CNN的缺点：
	只能处理静态数据，如果要处理时间序列、文本、语音、视频等动态数据，就要引入另一种神经网络，RNN，它可以说是当前大预言模型的鼻祖。

(2)编码
把文字变成机器能识别的向量，叫做编码。
独热编码 -> TF-IDF -> word2vec -> BGE

1)极端方法：只用一个数字代表一个字或者词。
	他 开心 我  就 生气
    45 89   441 3  8910
词表有多大，数字的范围就有多大。
缺点：维度太低。相当于一个一维向量。
	而且数字表示本身对语言理解没有任何意义。

2)另一种是准备一个高维数组，每个单词对应一个向量，这叫 one-hot 编码(独热编码)

缺点：太稀疏了。

	相当于每个维度就表示：是不是这个词？答案就是0和1。


(3)词嵌入 word embeding
维度太高不好，太低不好，折中一下，找个不高不低的维度。

通过词嵌入得到的向量，维度适中，

开心 [0 0    0.6  0   0    0.1]
我   [0 0.1  0    0   0    0.4]
就   [0 0    0    0   0.1  0]
生气 [0 0    0.6  0   0.1  0.4]



(4)余弦相似度: 看向量之间角度的大小
优点：可以计算单词之间的相关性。计算方式就是点积，就是余弦相似度。

a.b = Sigma(i=1, n, ai*bi)

cos(theta)=a.b/(|a|*|b|)


(5)词向量与嵌入矩阵
词向量:

这就将自然语言转化为可以用数学公式计算出来的方式。很关键！！

	每一行表示一个特征：类似Seurat的一行表示一个基因。
	总列数，叫此表大小。
	总行数，叫词向量维度。

	人类的认知局限在2d和3d，再高就想象不出来了。

(8)词向量的可视化

(9)可以输入神经网络中了

问题：输入太大了；

	类比：图像直接展开输入会损失像素之间的距离关联，使用卷积层解决了这个问题。

2.怎么体现自然语言的先后顺序？RNN
(1)简单情况: 褒贬预测
一次输入一个单词，输出+-1表示褒贬：
	他    X  -> g(W.X+b)  -> Y
第二个词进来，也是经过同样的网络：
	开心  X  -> g(W.X+b)  -> Y
使用<i>表示第几个单词：
	开心  X<2>  -> g(W.X<2>+b)  -> Y<2>
	这第二个单词完全没有利用第一个单词，没有体现前文的关联。

(2)怎么把前一个单词参与到当前单词的预测？

添加一个隐藏层 h<1>，
	* 然后该隐藏层用于预测Y<1>，
	* 同时该隐藏层 + 下一个单词的输入 一起预测第二个隐藏层:
示例:
	X<1>  -> g(Wxh.X<1>+bh)  
	                  ↓
	                  h<1>  -> g(Why.h<1> + by) -> Y<1>
	X<2>  -> g(Wxh.X<2> +Whh.h<1> +bh)
	                  ↓
	                  h<2>  -> g(Why.h<2> + by) -> Y<2>
注意区分权重矩阵: 
	* Wxh 表示由X 计算 隐藏层h的权重W。
	* Why 表示由h 计算 预测值y的权重W。
	* Whh 表示由h 计算 下一个隐藏层h的权重W。
	* 同理，偏置向量b也是两种，bh用于计算h，by用于计算y。

(3)循环神经网络 RNN，也有翻译作递归神经网络。
上图简化后：
	X<1>  --Wxh-> h<1> --Why-> Y<1>
	              ↓ Whh
	X<2>  --Wxh-> h<2> --Why-> Y<2>

还有这样画:
	X  ----Wxh--->   h   ----Why---> Y
	                ↑←↓ Whh
	表示中间的隐藏层h是上一个h参与生成的。

这就叫做RNN模型，它具备理解词和词之间关系的能力。
	可用于预测单词的褒贬、词性、给出一句话判断下一个字、翻译等多种自然语言处理能力。


(4)再展开讲讲权重矩阵: 注意维度
第一个单词的权重 Wxh 2x3 点乘 词向量X<1> 3x1 = 得到隐藏层h<1> 2x1;
	权重矩阵 Why 4x2 点乘 隐藏层 h<1> 2x1 = 第一个词的预测值 Y<1> 4x1;

(第二个单词的权重 Wxh 2x3 点乘 词向量X<2> 3x1) + (权重Whh 2x2 点乘 上一个隐藏层 h<1> 2x1) = 得到隐藏层h<2> 2x1;
	相当于上一个隐藏层 h<1> 2x1 拼接到 单词X<2> 3x1的上部，得到维度为 5x1;
	Whh 2x2 拼接到 Wxh 2x3 左侧，得到维度 2x5
	两个扩展向量点乘，得到输出 h<2> 2x1

后面流程就一样了，单词和上一个隐藏层 共同生成新隐藏层。
	该隐藏层用于预测该单词的预测值。
	并用于生成下一个隐藏层。


(5)会看公式

h<t> = g1(Wxh.X<t> + Whh.h<t-1> + b1)
y<t> = g2(Why.h<t> + by)

和经典的神经网络比，就是多了一个前一个时刻的隐藏状态。


小结:
	要预测单词，就需要让计算机读懂单词，需要对词做编码。
	词编码后的向量，就是词向量。
	编码词向量的方法：
	* one-hot 独热编码：只有0和一个1。维度太高，太稀疏。
	* 词嵌入 word embeding: 是用神经网路训练得到的。
		词嵌入后得到的矩阵，叫做 嵌入矩阵。
		词向量所在的空间，叫潜空间。
		词向量之间的相关性，可用余弦相似度，或者点积来计算。
	有了词向量，即可输入到神经网络。
	经典的nn无法表达词的先后顺序，因此增加隐藏层h<i>。





3.RNN的问题
(1)无法捕捉长期依赖
随着步骤的增多，之前的信息会逐步丢失，无法捕捉长期依赖。
而有的语句，恰恰是距离很远的铺垫在起到关键作用。

(2)无法并行计算
RNN必须按顺序处理，每个时间步骤依赖上一个时间的隐藏状态计算结果，无法并行计算。

	- 输出门
	- 隐藏状态

|-- Transformer 架构
----------------------------------------
与基于RNN的方法相比，Transformer 不需要循环，主要是由Attention 机制组成，因而可以充分利用python的高效线性代数函数库，大量节省训练时间。


谷歌提出的transformer 架构，其本质上是一个Encoder-Decoder的结构。把英文句子输入模型，模型会输出法文句子。

要搭建Transformer，我们必须要了解5个过程:
	词向量层 / 词嵌入
	位置编码
	创建Masks
	多头注意层（The Multi-Head Attention layer）
	Feed Forward层
	归一化
	MLP 多层感知机
	softmax

GPT: 只有decoder，不需要翻译，只需要知道下一个词是什么即可。
	控制随机性的参数：温度 Temperature; 随机性太高容易胡说八道，太低容易保守。
	控制范围从概率最高的k个词选择：Top-K
	只在语言上流畅，但在逻辑上、事实上狗屁不通的现象：幻觉 Hallucination
	检索增强生成 RAG(Retrieval-Augmented Generation): 使用私有数据库，检索后生成。
	知识库 KB(Knowledge Base): 私有的数据库。
	向量数据库 Vector Database: 知识通常会以向量形式保存在数据库中。
	量化 Quantization: 使用低浮点数，降低显存。
	蒸馏 Dustillation: 使用参数较大的大模型指导参数较小的小模型。
	剪枝 Pruning: 删除模型中不重要的神经元，让模型更稀疏，以提高速度的操作。


1.翻译问题
From: I love you baby.
to: 我 爱 你 宝贝。

步骤
(1)先用词嵌入方式，把词转为词向量。
简单点，假设维度是6:
I    [0 0 0 0 0.12  0.15].T
love [0 0 0 0 0.92  0.85].T
you  [0 0 0 0 0.13  0.25].T
bady [0 0 0 0 0.52  0.5].T

(2)普通NN
每个词向量都丢到全连接神经网络中，则都没有上下文信息。
词向量 -> FC -> Y1

(3)使用RNN
I -> h1 -> love -> h2 ->you ->h3 -> baby -> h4
     ↓             ↓          ↓             ↓
     Y1            Y2         Y3            Y4
又面临串行计算，句子太长时长期记忆困难的问题。




2.全新机制：attention
(1)位置编码：表示该词出现在句子中的位置
给每个词向量 + 位置编码。维度不变(6x1)，但是包含了位置信息。

(2)现在每个词向量还没有 其他词的信息
使用矩阵 Wq(6x6) 点乘 向量 I (6x1) = q1 向量(6x1)
	其中Wq矩阵是一组可以通过训练获得的一组权重值。

使用矩阵 Wk(6x6) 点乘 向量 I (6x1) = k1 向量(6x1)
使用矩阵 Wv(6x6) 点乘 向量 I (6x1) = v1 向量(6x1)

这样，就获得了 词向量 I 的三个向量：q1,k1,v1;
	对于其他几个单词 i，也类似获得自己的 qi,ki,vi;

(3)实际计算时，是通过GPU矩阵计算的:
	Wq矩阵 点乘 词向量构成的矩阵 = Q 矩阵(q列向量)
	Wk矩阵 点乘 词向量构成的矩阵 = K 矩阵
	Wv矩阵 点乘 词向量构成的矩阵 = V 矩阵
本文为了解释方便，还是分开按q列向量说。


(4)第一个词向量和其他词的相似度 q.k=向量相似度
现在原来的每个词向量，已经通过线性变换映射为 qi, ki, vi 三个同维度列向量。

q1 点积 k2 = a12，表示在第一个词视角中，1,2两个词的相似度。
q1 点积 k3 = a13，表示在第1个词视角中，1,3两个词的相似度。
q1 点积 k4 = a14，表示在第1个词视角中，1,4两个词的相似度。
q1 点积 k1 = a11，表示在第1个词视角中，1,1两个词的相似度，和自己相似度最大。


(5)相似度和v列向量相乘，再求和，得到a1，包含了词向量1的全部上下文信息。
a11.v1 + a12.v2 + a13.v3 + a14.v4 = a1

表示在词向量1视角下，按照与词向量1相似度作为权重，把其余词向量加权求和。

这样就把上下文信息都包含在向量a1中了。

同理，可以获得 a2, a3, a4。都表示在各自视角下的上下文信息。

现在，最原始的词向量，已经转化为包含位置信息+上下文信息的新向量，维度不变。

这就是 attention 机制做的事情了。


(6)进一步优化: 多头注意力机制
很多事情从不同角度看，结果是不同的。
所以可以尝试多次计算attention，把结果拼接起来。

之前每个词向量计算出一个q,k,v向量，现在计算出两组q,k,v，给每个词两次学习机会。
	以增加语言的灵活性。
	这里每个q,k,v称为一个头: head1, head2
	这里维度为什么小了？词向量 6x1，q/k/v 向量 3x1，难道是Wq小了: 3x6 . 6x1 = 3x1
		权重Wq是什么？怎么来的？

接着，每个头的q,k,v，依旧经过上述注意力层的运算，得到a向量: a11, a12 (3x1)

然后两个a向量拼接起来，得到向量a1 (6x1)
	这就叫做 多头注意力 Multi-Head Attention.
	以上就是2头的例子。

以上就是 transformer 最核心的 attention 机制。




3.transformer 架构
(1)输入的input 通过词嵌入方式，转换为词向量矩阵。
(2)加入位置信息：其实就是 加上 一个形状一样的矩阵向量。
(3)多头注意力：输出的矩阵和输入的没有变化。
	给每个词向量增加了上下文信息。
	三个多头注意力机制：输入，输出，结合。
(4)添加残差网络 和 归一化处理。
是为了解决梯度消失，并且让分布更加稳定而做的优化。

(5)深入到多头注意机制的细节 原论文Fig2
单头注意力：
	- Q.K 得到相似度系数矩阵;
	- 再和V相乘，得到包含上下文信息的词向量矩阵。

	- 中间省略了3步：缩放 scale，掩码 Mask，softmax处理。

多头注意力：
	- 首先Q/K/V经过线性变换，拆分成多组，就叫多头。
		* 相当于给了多次机会，学习到不同的相似度关系。
	- 依次经过 单头注意力运算后
	- 把结果拼接起来 Concat
	- 省略有最后的线性变换：多头结果不是简单拼接起来，还需要经过一层权重矩阵的乘法。

(6)这时再看论文中的两个核心公式1和2
所谓注意力运算，就是:
	Q.K^T 运算，经过缩放(除以 sqrt(dk) );
	取 softmax;
	再和V做矩阵乘法。

多头注意力机制：
	就是Q/K/V矩阵经过多个权重矩阵Wqi,Wki,Wvi拆分到多个头中，分别经过注意力机制运算(where 子句);
	最后合并起来；
	在经过一次矩阵计算Wo，得到输出。

(7)再看全局图
左边叫 编码器 Encoder；右边叫解码器 Decoder；

首先输入要翻译的文本：I love you baby.
	经过词嵌入，引入位置编码，
	经过多头注意力，残差和归一化层，接着送入全连接NN，再残差和归一化处理。 重复N次。
	结果送入右侧解码器的多头注意力机制的2个输入中，作为K和V矩阵。

再看右侧解码器部分：
	输出是翻译后的文本：我 爱 你 宝贝
	同样经过词嵌入，引入位置编码，
	经过多头注意力，残差和归一化层，接着送入多头注意力机制的1个输入中，作为Q矩阵。
		经过一个 残差和归一化层，全连接NN，再残差和归一化处理。 重复N次。
	在经过一层线性变换的神经网络，投射到词表向量中。
	最后用softmax层转化为概率，这就是下一个词在词表中的概率分布。
	取概率最高的单词，就是下一个词应该是什么。
	
注意：右侧有一个掩码，是因为训练时，不应该看到下文，模拟真实推理场景。
	比如，输入“我”，看不到下文，预测是“爱”，
	然后输入 “我爱”，看不到下文，预测是“你”，
	等等。
	如果预测的不对，就计算损失函数，再反向传播，调整transformer结构中的各种权重矩阵，直到学习好未知。


(8)总的来说，transformer 确实是一个特别简单的架构(We propose a new simple network architecture)
相对于CNN，只有多头注意力这一层是陌生的。
	但是拆解后，它也是各种矩阵相乘，相加的普通操作。

其余的概念，都是之前已经了解过的概念：
	词嵌入
	位置编码
	残差
	归一化
	经典神经网络中的 MLP(多层感知机)
	softmax 层

GPT 的底层，只有一半，即 decoder 部分。

1. 无意义实例
我们以最简单的网络定义来学习pytorch的基本使用方法，我们接下来要定义一个神经网络，包括一个输入层，一个隐藏层，一个输出层，这些层都是线性的，给隐藏层添加一个激活函数Relu，给输出层添加一个Sigmoid函数

import torch
import torch.nn as nn

class SimpleNet(nn.Module):
    def __init__(self):
        super(SimpleNet, self).__init__()
        self.fc1 = nn.Linear(1, 32)
        self.relu = nn.ReLU()
        self.fc2 = nn.Linear(32, 1)
        self.Sigmoid = nn.Sigmoid()

    def forward(self, x):
        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)
        x = self.Sigmoid(x)
        return x


(2)模型编译
我们在之前的机器学习文章中反复提到过，模型的训练是怎么进行的呢，要有一个损失函数与优化方法，我们接下来看看在pytorch中怎么定义这些

import torch.optim as optim

# 实例化模型对象
model = SimpleNet()
# 定义损失函数
criterion = nn.MSELoss()
 
# 定义优化器
learning_rate = 0.01
optimizer = optim.SGD(model.parameters(), lr=learning_rate)

我们上面创建的神经网络是一个类，所以我们实例化一个对象model，然后定义损失函数为mse，优化器为随机梯度下降并设置学习率


(3) 模型训练
# 创建随机输入数据和目标数据
input_data = torch.randn((100, 1))  # 100个样本，每个样本有1个特征
target_data = torch.randn((100, 1))  # 100个样本，每个样本有1个目标值
 
# 训练模型
epochs = 100
 
for epoch in range(epochs):
    # 前向传播
    output = model(input_data)
 
    # 计算损失
    loss = criterion(output, target_data)
 
    # 反向传播和优化
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

以上步骤是先创建了一些随机样本，作为模型的训练集，然后定义训练轮次为100次，然后前向传播数据集，计算损失，再优化，如此反复

    Dense(32, input_dim=1, activation='relu'),
    Dense(1, activation='sigmoid')
])

注意这里也是一层输入层，一层隐藏层，一层输出层，和pytorch一样，输入层是隐式的，我们的输入数据就是输入层，上述代码定义了一个隐藏层，输入维度是1，输出维度是32，还定义了一个输出层，输入维度是32，输出维度是1，和pytorch环节的模型结构是一样的

(2) 模型编译
那么在Keras中模型又是怎么编译的呢

model.compile(loss='mse', optimizer='sgd')

非常简单，只需要这一行代码 ，设置损失函数为mse，优化器为随机梯度下降


(3) 模型训练
模型的训练也非常简单

# 创建随机输入数据和目标数据
import numpy as np
input_data = np.random.randn(100, 1)  # 100个样本，每个样本有10个特征
target_data = np.random.randn(100, 1)  # 100个样本，每个样本有5个目标值

# 训练模型
model.fit(input_data, target_data, epochs=100)

因为我们已经编译好了损失函数和优化器，在fit里只需要输入数据，输出数据和训练轮次这些参数就可以训练了
